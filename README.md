### Uso de par√©ntesis

Python soporta todos los ![operadores matem√°ticos](assets/videos/Python-Operadores%20AritmeÃÅticos.mp4 "Operadores Aritm√©ticos") m√°s comunes, conocidos como operadores aritm√©ticos. ```

El uso de par√©ntesis no solo se aplica a los operadores aritm√©ticos, sino que tambi√©n pueden ser aplicados a otros operadores como los **relacionales o de membres√≠a**. En Python, los operadores relacionales o de membres√≠a se utilizan para comparar valores y evaluar condiciones booleanas. Estos operadores se definen como sigue:

- == (igual que): eval√∫a si el valor del lado izquierdo es igual al valor del lado derecho.
- != (diferente): eval√∫a si el valor del lado izquierdo es diferente al valor del lado derecho.
- > (mayor que): eval√∫a si el valor del lado izquierdo es mayor que el valor del lado derecho.
- < (menor que): eval√∫a si el valor del lado izquierdo es menor que el valor del lado derecho.
- > = (mayor o igual que): eval√∫a si el valor del lado izquierdo es mayor o igual que el valor del lado derecho.
- <= (menor o igual que): eval√∫a si el valor del lado izquierdo es menor o igual que el valor del lado derecho.

Estos operadores se utilizan com√∫nmente en sentencias `if` y `elif` para tomar decisiones basadas en condiciones espec√≠ficas. Por ejemplo:

```Python
x = 5
if x > 3:
    print("x es mayor que 3")
elif x == 5:
    print("x es igual a 5")
else:
    print("x es menor que 3")
```

En el anterior ejemplo, se eval√∫a la condici√≥n x > 3 y, si es verdadera, se imprime el mensaje correspondiente. Si no es verdadera, se eval√∫a la condici√≥n x == 5 y, si es verdadera, se imprime otro mensaje. Si ninguna de las condiciones es verdadera, se ejecuta el bloque `else`.

Adem√°s, los operadores relacionales tambi√©n se utilizan en expresiones que asignan valores booleanos, como en un ejemplo simple:


En este caso, el operador `<=` eval√∫a la condici√≥n y devuelve un valor booleano (True en este caso). Este valor booleano se puede utilizar en sentencias `if` o en otras expresiones.

Recuerda que los operadores relacionales son estrictos, lo que significa que 4 no es considerado igual a 4.0, por ejemplo. Si se necesita comparar valores num√©ricos con precisi√≥n decimal, se debe utilizar bibliotecas adicionales como `decimal` o `fractions` que vendr√≠a a ser otro tema de presentaci√≥n m√°s adelante.

## Paso por valor y por referencia

As√≠ mismo, en Python, hay dos maneras en que las funciones pueden tratar los par√°metros de entrada: paso por valor y paso por referencia.

**1. Paso por valor**: Los tipos simples como enteros, flotantes, cadenas y l√≥gicos se pasan por valor. Esto significa que se crea una copia local de la variable dentro de la funci√≥n, por lo que cualquier modificaci√≥n realizada en la funci√≥n no afecta la variable original. Ver el siguiente ejemplo:

```Python
def incrementar(x): # funci√≥n llamada incrementar
    x = x + 1 # variable x

x = 5
incrementar(x)
print(x)  # Output: 5
```

En el anterior ejemplo, la variable `x` se pasa por valor a la funci√≥n incrementar. La modificaci√≥n realizada en la funci√≥n `(x = x + 1)` no afecta la variable original `x`, que sigue siendo 5.

**2. Paso por referencia**: Los tipos compuestos como listas, [diccionarios](https://devcode.la/tutoriales/diccionarios-en-Python/) y objetos se pasan por referencia. Esto significa que se env√≠a una copia de la referencia a la variable original, por lo que cualquier modificaci√≥n realizada en la funci√≥n afecta la variable original. Ver el siguiente ejemplo:

```Python
def agregar_elemento(numeros): # Funci√≥n llamada agregar_elemento y variable numeros
    numeros.append(4)

numeros = [1, 2, 3]
agregar_elemento(numeros)
print(numeros)  # Output: [1, 2, 3, 4]
```

En el anterior ejemplo, la variable numeros se pasa por referencia a la funci√≥n `agregar_elemento`. La modificaci√≥n realizada en la funci√≥n (`numeros.append(4)`) afecta la variable original `numeros`, que ahora contiene cuatro elementos.

Para evitar que los tipos compuestos se modifiquen accidentalmente, se puede crear una copia de la lista antes de pasarla como par√°metro:

```Python
def agregar_elemento(numeros):
    numeros.append(4)

numeros = [1, 2, 3]
agregar_elemento(numeros[:])  # Crear copia con slicing
print(numeros)  # Output: [1, 2, 3]
```

En este ejemplo, se crea una copia de la lista `numeros` utilizando [slicing](https://naps.com.mx/blog/listas-en-Python/) (`numeros[:]`), y se pasa esa copia como par√°metro a la funci√≥n `agregar_elemento`. La variable original `numeros` no se modifica.

## Variable global en Python

En Python, las variables globales son aquellas definidas en el cuerpo principal del programa, fuera de cualquier funci√≥n. Estas variables son accesibles desde cualquier punto del programa, incluyendo dentro de funciones. Tambi√©n se puede acceder a las variables globales de otros m√≥dulos o programas importados.

Por otro lado, las variables locales son aquellas definidas dentro de una funci√≥n y solo permiten su acceso desde ella. No se pueden acceder o modificar desde fuera de la funci√≥n en la que se definen.

Para acceder y modificar variables globales, se puede utilizar la funci√≥n `globals()`, que devuelve un diccionario que contiene todas las variables del √°mbito global actual. Este diccionario se utiliza para acceder al valor de las variables globales y modificarlo. Sin embargo, para variables locales, se utiliza la funci√≥n `locals()`, que devuelve un diccionario que contiene todas las variables del √°mbito local actual, pero no se puede modificar.

A continuaci√≥n, se presenta un ejemplo que ilustra la diferencia entre variables globales y locales en Python:

```Python
# Variable global llamada global_valor
global_valor = 10

def funcion(): # Funci√≥n llamada funcion
    # Variable local llamada local_valor
    local_valor = 20
    print(f'La variable local valor es {locals()["local_valor"]}')
    print(f'La variable global valor es {globals()["global_valor"]}')

    # Modificamos la variable global
    globals()['global_valor'] = 30

funcion()
print(f'La variable global valor es ahora {global_valor}')
```

En el anterior ejemplo, se define una variable global llamada `global_valor` y una variable local llamada `local_valor` dentro de la funci√≥n llamada `funcion`. Luego, se modifica el valor de la variable global utilizando el diccionario devuelto por `globals()`. Al final, se imprime el valor actual de la variable global `global_valor`, que ha sido modificada dentro de la funci√≥n.

Es importante destacar que las variables locales no se pueden acceder ni modificar desde fuera de la funci√≥n en la que se definen, mientras que las variables globales pueden ser accedidas y modificadas desde cualquier punto del programa

## Condicionales

Las [sentencias condicionales](https://keepcoding.io/blog/expresiones-condicionales-en-python/) son un aparte fundamental en la programaci√≥n en Python; como, en cualquier lenguaje de programaci√≥n; por lo general, son una forma de controlar el flujo de ejecuci√≥n de un programa, permitiendo que el c√≥digo se ejecute solo cuando se cumplan ciertas condiciones establecidas. A continuaci√≥n, se presentan algunos de los conceptos b√°sicos y unos ejemplos de c√≥mo utilizarlas:

```python
if expresion_booleana:
     # C√≥digo a ejecutar si la expresi√≥n es verdadera
else:
     # C√≥digo a ejecutar si la expresi√≥n es falsa
```

- La sentencia condicional m√°s b√°sica en Python es la sentencia `if`, que se expresa de la siguiente forma: `if condici√≥n: c√≥digo a ejecutar`. La condici√≥n se eval√∫a como verdadera o falsa, y si es verdadera, se ejecutan las l√≠neas de c√≥digo indentadas debajo de la sentencia `if`.
- Para especificar un c√≥digo alternativo que se ejecute cuando la condici√≥n sea falsa, se utiliza la sentencia `else: if condici√≥n: c√≥digo a ejecutar elif condici√≥n2: c√≥digo a ejecutar else: c√≥digo a ejecutar por defecto`.
- La sentencia `elif` (short for ‚Äúelse if‚Äù) se utiliza para evaluar otra condici√≥n si la primera condici√≥n es falsa. De esta forma, se pueden verificar m√∫ltiples condiciones y ejecutar diferentes bloques de c√≥digo seg√∫n sea necesario. por ejemplo:

```python
x = 5

if x > 10:
    print("x es mayor que 10")
elif x == 5:
    print("x es igual a 5")
else:
    print("x es menor que 10")
```

## algunos Consejos

- Es importante tener en cuenta que las sentencias condicionales deben estar indentadas correctamente para que Python las interprete como parte del bloque de c√≥digo asociado.
- A menudo, se utilizan operadores l√≥gicos (como `and`, `or`) y de comparaci√≥n (como `==`, `!=`, `>`, `<`, etc.) para definir las condiciones. Por ejemplo: `if x > 5 and y < 10:`
- Para evitar errores y mantener el c√≥digo legible, se recomienda **_evitar la anidaci√≥n excesiva_** de condicionales y **_utilizar la indentaci√≥n adecuada_**para definir los bloques de c√≥digo.

## Evaluaci√≥n en corto circuito

Normalmente, la Evaluaci√≥n en Corto Circuito ([ECC](https://patriciaemiguel.com/conceptos/2020/04/20/circuito-corto.html)) dentro de Python hace referencia a la forma en que las operaciones l√≥gicas (Booleanas) se eval√∫an cuando se utilizan **operandos booleanos** dentro de una condici√≥n condicional. O sea que, cuando se combinan operadores l√≥gicos como `and` u `or`, Python eval√∫a los operandos de **_izquierda a derecha_** y por lo tanto corta o termina la evaluaci√≥n tan pronto como se determine la verdad (true) o falsedad (false) de la condici√≥n. Por favor ver la siguiente ilustraci√≥n 9.

![Corto Circuito](assets/img/Evaluaci√≥nPorCortoCircuito.png "Evaluaci√≥n por corto circuito")
Fuente: [https://patriciaemiguel.com/conceptos/2020/04/20/circuito-corto.html](https://patriciaemiguel.com/conceptos/2020/04/20/circuito-corto.html)

En el siguiente ejemplo, `if (A and B)`, si `B` es nulo (o falso en un sentido l√≥gico), Python evaluar√° directamente `B` como falso (false) por lo que no continuar√° evaluando el resto de la condici√≥n, ya que el resultado de una operaci√≥n `and` con un falso (false) operando es siempre falso (false).

Es as√≠ como se conoce el termino como **cortocircuito a la izquierda** (Left-Hand Side Short-Circuit) para operaciones `and`, ya que Python ‚Äúcorta‚Äù la evaluaci√≥n cuando encuentra el primer falso (false ) en el operando.

Dicho de otro modo, la Evaluaci√≥n en Corto Circuito en Python es una caracter√≠stica que permite en la mayor√≠a de los casos evitar errores al evaluar condiciones condicionales, ayudando a que, solo se eval√∫en los operandos necesarios para determinar el resultado final. Obs√©rvese el siguiente ejemplo:

```python
def ejemplo(A, B):
    if A and B:  # Cortocircuito a la izquierda (LHSSC) para operaciones and
        print("Ambos operandos son verdaderos")
    else:
        print("Al menos uno de los operandos es falso")

# Ejemplo de uso:
A = True
B = None  # B es nulo, por lo que se considera falso en un sentido l√≥gico
ejemplo(A, B)  # Output: "Al menos uno de los operandos es falso"
```

Analizando el anterior ejemplo, como `B` es nulo, Python eval√∫a directamente `B` como falso y no continua evaluando el resto de la condici√≥n, oprimiendo el mensaje correspondiente.
